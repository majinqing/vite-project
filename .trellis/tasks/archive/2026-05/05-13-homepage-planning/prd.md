# brainstorm: homepage planning

## Goal

为首页补充清晰、合适的内容结构与视觉方向，在确认范围后再决定是否继续实现或调整现有首页代码，避免脱离需求直接开发。

## What I already know

* 用户明确要求这次按 Trellis 流程先梳理需求，再决定是否动代码。
* 首页路由当前指向 `src/views/index/index.vue`。
* 应用整体使用 `Vue 3 + TypeScript + Vite + Element Plus + SCSS`。
* 现有布局壳子、左侧菜单和顶部导航已存在，首页属于 `src/views/index/` 下的路由页面。
* 当前首页代码已经被我替换成一版静态仪表盘样式，但这版实现未经需求确认，不应视为最终方向。
* 项目场景与文案倾向于“新疆数字棉花科技创新平台 / 物联网设备管理”这一类农业数字化平台。

## Assumptions (temporary)

* 本次任务优先聚焦首页的信息架构和展示重点，而不是先接真实接口。
* 首页大概率需要服务两类用户：值班/运营人员，以及管理查看人员。
* 短期内更适合先做 MVP：静态结构或半静态结构，后续再逐步绑定真实数据与跳转。

## Open Questions

* 无。

## Requirements (evolving)

* 本轮首页采用“内容型 MVP”。
* 首页需要先把信息架构定清楚，做成完整静态或半静态页面。
* 本轮首页聚焦基础模块：欢迎区、建设成效指标、成果摘要、公告动态、快捷入口。
* 本轮不强绑定真实接口数据，但需要为后续接入预留扩展点。
* 首页主要服务管理查看场景，信息排序偏“管理概览型”。
* 首页第一屏核心叙事偏“平台建设成效”，优先体现平台覆盖、建设成果与总体能力。
* 本轮仅做当前需求，不额外扩展专题分析、地图总览或真实业务入口规划。
* 首页顶部第一层先告诉管理者“这是什么平台、现在看什么”，承担平台定位与导览作用。
* 首页第二层放建设成效指标，直接承接第一屏叙事，用核心数字证明平台建设成果。
* 首页第三层放成果摘要，用摘要卡片解释建设成果、覆盖场景或阶段亮点。
* 首页第四层放快捷入口，提供设备管理、报表查看、专题入口等常用行动区。
* 首页第五层放公告动态，作为补充信息区，不打断主叙事。

## Acceptance Criteria (evolving)

* [x] 首页范围和目标明确，形成可执行的 MVP 定义。
* [x] 明确首页主要服务对象，以及模块优先级。
* [x] 明确管理概览首页的核心叙事重点。
* [x] 明确是否需要为后续扩展预留重点模块。
* [x] 明确哪些内容属于本轮范围，哪些内容延后。
* [x] 明确第四层与第五层模块顺序。
* [x] 在用户确认前，不把现有临时代码当作最终方案推进。

## Technical Approach

本轮先输出首页信息架构确认稿，再决定是否进入实现。若进入实现，首页将采用静态或半静态内容组织，围绕“平台定位 -> 建设成效指标 -> 成果摘要 -> 快捷入口 -> 公告动态”的层级展开，保留后续替换为真实数据和真实跳转的空间，但本轮不接接口。

## Definition of Done (team quality bar)

* 需求范围已确认并写入本文件。
* 若进入实现阶段，遵循 `.trellis/spec/frontend/` 规范。
* 若发生行为变更，完成构建验证。

## Out of Scope (explicit)

* 本轮不直接接入复杂后端接口。
* 本轮不要求完成真实接口联调。
* 本轮不额外扩展专题分析、地图总览或真实业务入口规划。
* 未经确认扩展到多个页面或整个设计系统重构。
