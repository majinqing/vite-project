# 物联网设备管理新增接入测试用例

## 测试对象

- `src/views/iot/equipment/index.vue`

## 前置条件

- 页面组件可正常渲染
- 测试环境支持替代 Element Plus 基础输入和按钮组件
- 页面设备数据当前使用前端 mock 列表

## 核心场景

- 初始渲染时，新增接入弹窗默认隐藏
- 点击“新增接入”后，出现录入弹窗
- 填写模拟数据并提交后，设备列表新增一条记录
- 新增记录可以被全局搜索命中
- 点击取消后，弹窗关闭且不会新增记录
- 页面内容区域可以完整渲染，不依赖固定高度裁切

## 异常与边界场景

- 未完成必填字段时，点击提交不应新增设备
- 提交新增后再次打开面板，表单应恢复默认空值
- 提交新增时若当前存在筛选条件，应保证新增后用户能重新定位该设备

## 回归检查点

- 原有全局搜索、关键字筛选、区域筛选、状态筛选继续可用
- 原有设备台账、告警跟进、区域运行布局不被新增弹窗破坏
- 重置筛选依然能恢复默认状态

## 用例步骤与预期结果

1. 初始渲染页面  
预期结果：看不到新增接入弹窗；设备列表默认展示全部设备；页面底部核心区域可继续向下展示

2. 点击“新增接入”  
预期结果：出现录入弹窗，显示设备名称、设备类型、所属区域、接入协议、负责人等字段

3. 填写模拟数据并点击“确认接入”  
预期结果：弹窗关闭；设备列表新增设备；新设备名称出现在页面中

4. 在顶部全局搜索中输入刚新增的设备名称  
预期结果：列表能够筛选出该新增设备

5. 点击“新增接入”后直接点击“取消”  
预期结果：弹窗关闭；设备列表数量保持不变
