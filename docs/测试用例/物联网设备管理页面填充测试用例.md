# 物联网设备管理页面填充测试用例

## 测试对象

- `src/views/iot/equipment/index.vue`

## 前置条件

- 页面路由组件文件存在且可正常加载
- 测试环境可挂载 Vue 组件并渲染 Element Plus 组件
- 页面使用静态 mock 数据作为展示来源

## 核心场景

- 页面初始渲染时，显示中文标题、概览指标和默认设备列表
- 输入关键字后，列表按设备名称、编号或类型进行过滤
- 选择设备状态后，仅展示匹配状态的设备
- 点击“重置筛选”后，列表恢复默认全量展示

## 异常与边界场景

- 输入不存在的关键字时，列表应显示为空
- 同时使用区域筛选和状态筛选时，结果应按组合条件过滤
- 重置筛选时，无论此前执行过何种组合筛选，都应恢复默认值

## 回归检查点

- 页面仍保留总览区、告警区和区域运行区，不因列表筛选逻辑被破坏
- 页面中的关键中文文案仍可正常显示
- 设备状态标签与筛选值保持一致，不出现在线/预警/离线错配

## 用例步骤与预期结果

1. 初始渲染页面
预期结果：显示“物联网设备管理”标题；显示概览指标卡；默认列表展示全部设备

2. 在搜索框输入“网关”
预期结果：列表仅保留名称、编号或类型中包含“网关”的设备

3. 将状态切换为“预警”
预期结果：列表仅显示状态为“预警”的设备

4. 先选择区域“南区示范带”，再选择状态“离线”
预期结果：列表仅保留同时满足区域和状态条件的设备

5. 点击“重置筛选”
预期结果：搜索框清空、筛选项回到默认值、设备列表恢复为全量数据
