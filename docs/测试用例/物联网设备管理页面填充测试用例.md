# 物联网设备管理页面填充测试用例

## 测试对象

- `src/views/iot/equipment/index.vue`

## 前置条件

- 页面路由组件文件存在且可正常加载
- 测试环境可挂载 Vue 组件并替代 Element Plus 组件
- 页面当前使用静态 mock 数据作为展示来源

## 核心场景

- 页面初始渲染时，显示顶部中文标题、全局搜索框、概览指标和默认设备列表
- 在顶部全局搜索中输入关键字后，设备列表按设备名称、编号或类型进行过滤
- 全局搜索输入后，设备台账中的关键字筛选框同步显示相同内容
- 选择设备状态后，仅展示匹配状态的设备
- 点击“重置筛选”后，顶部全局搜索、关键字、区域、状态全部恢复默认

## 异常与边界场景

- 输入不存在的关键字时，设备列表应显示为空
- 同时使用全局搜索、区域筛选和状态筛选时，结果应按组合条件过滤
- 通过设备台账关键字筛选修改条件后，顶部全局搜索也应同步更新

## 回归检查点

- 页面仍保留总览区、告警区和区域运行区，不因新增顶部搜索而破坏原有布局
- 设备状态标签与筛选值保持一致，不出现在线、预警、离线错配
- 重置筛选后，不残留顶部搜索关键字或设备台账关键字

## 用例步骤与预期结果

1. 初始渲染页面  
预期结果：显示“物联网设备管理”标题；显示顶部全局搜索框；默认列表展示全部设备

2. 在顶部全局搜索框输入“网关”  
预期结果：列表仅保留名称、编号或类型中包含“网关”的设备；设备台账关键字输入框同步显示“网关”

3. 将状态切换为“预警”  
预期结果：列表仅显示状态为“预警”的设备

4. 先在顶部输入“终端”，再选择区域“南区示范带”，再选择状态“离线”  
预期结果：列表仅保留同时满足关键字、区域、状态条件的设备

5. 点击“重置筛选”  
预期结果：顶部搜索框清空；设备台账关键字输入框清空；区域和状态回到默认值；设备列表恢复为全量数据
