# 物联网设备管理新增接入

## 需求背景

物联网设备管理页面当前已经有设备总览、筛选和台账展示，但“新增接入”按钮还没有实际行为。为了先把业务链路跑通，需要补一个前端本地可用的新增接入 MVP，让页面在不依赖后端接口的情况下完成设备录入和列表联动。

## 目标与价值

- 补齐“新增接入”按钮的最小可用交互
- 让用户可以在页面内录入模拟设备并立即看到结果
- 让新增设备直接复用现有搜索和筛选链路
- 为后续接入真实接口预留页面结构和前端状态流

## 功能范围

- 点击“新增接入”按钮后显示设备录入弹窗
- 支持录入设备名称、设备类型、所属区域、接入协议、负责人
- 提交后基于模拟数据生成新设备记录并插入设备列表
- 新增后设备可参与全局搜索、关键字筛选、区域筛选、状态筛选
- 支持取消新增接入并关闭弹窗
- 优化页面高度与滚动样式，确保总览、台账、告警和区域运行内容完整展示

不在本次范围内：

- 不接真实后端接口
- 不实现编辑、删除、批量导入
- 不实现复杂校验、文件上传、地图选点

## 用户操作流程

1. 用户进入“物联网设备管理”页面。
2. 用户点击设备台账区域右上角“新增接入”按钮。
3. 页面弹出新增接入录入弹窗，展示基础录入字段。
4. 用户填写模拟设备信息并点击“确认接入”。
5. 页面生成一条新的模拟设备记录，并插入设备台账。
6. 用户可以继续通过全局搜索或筛选条件定位刚新增的设备。
7. 如果用户点击“取消”，弹窗关闭且不新增任何设备。

## 验收标准

- 页面初始状态下不显示新增接入弹窗。
- 点击“新增接入”后，能看到设备录入弹窗和基础字段。
- 提交一组完整模拟数据后，设备列表增加新设备记录。
- 新增设备可被顶部全局搜索和设备台账关键字筛选命中。
- 点击“取消”后，弹窗关闭且设备列表数量不变。
- 再次打开新增接入弹窗时，表单恢复默认空值。
- 页面主体区域在桌面尺寸下不再因为固定高度裁切底部内容，用户可完整查看设备台账、告警跟进和区域运行。

## 待确认项 / 风险点

- 当前只做前端 mock 数据新增，刷新页面后不会持久化。
- 后续接入真实接口时，需要重新定义设备编号生成、默认状态、提交成功反馈和失败处理。
- 若后续弹窗字段继续扩展，需要决定是否引入表单校验提示与提交中状态。
