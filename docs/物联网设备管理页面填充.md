# 物联网设备管理页面填充

## 需求背景

当前物联网设备管理页已经有设备总览、告警跟进、区域运行和设备台账内容，也具备设备列表筛选能力，但关键字筛选入口位于设备台账区域。用户进入页面后，如果想直接搜索目标设备，需要先滚动到列表区域，操作路径偏长。

本次希望在页面最上方补一个全局搜索入口，让用户进入页面后就能直接搜索设备，并复用现有筛选逻辑。

## 目标与价值

- 为物联网设备管理页补齐更直观的顶部搜索入口
- 降低用户从页面顶部定位目标设备的操作成本
- 复用现有关键字筛选逻辑，避免新增独立状态和重复规则
- 保持页面“总览 + 告警 + 台账”的信息结构不变

## 功能范围

- 在页面头部总览区新增全局搜索框
- 全局搜索支持按设备名称、编号、类型检索设备
- 全局搜索与设备台账中的关键字筛选共用同一状态，任一处输入都能同步另一处
- 设备区域筛选、状态筛选、重置筛选继续沿用现有行为

不在本次范围内：

- 不接入真实后端搜索接口
- 不新增协议、负责人、电量等其他筛选维度
- 不调整路由结构或菜单契约

## 用户操作流程

1. 用户进入“物联网设备管理”页面。
2. 页面顶部默认展示总览信息和全局搜索框。
3. 用户在顶部输入设备名称、编号或类型关键字后，设备列表立即按条件过滤。
4. 用户滚动到设备台账区域时，可以看到关键字筛选项已同步当前搜索内容。
5. 用户可继续叠加区域筛选、状态筛选缩小范围。
6. 用户点击“重置筛选”后，顶部全局搜索、关键字筛选、区域、状态全部恢复默认值。

## 验收标准

- 进入 `/iot/equipment` 页面时，页面最上方可见全局搜索输入框。
- 顶部全局搜索可根据设备名称、编号或类型过滤设备列表。
- 顶部全局搜索与设备台账关键字筛选保持双向同步。
- 区域筛选、状态筛选仍可与关键字搜索组合生效。
- 点击“重置筛选”后，所有筛选条件恢复默认状态。
- 页面在桌面和较窄宽度下均能自然排版，不出现明显遮挡或错位。
