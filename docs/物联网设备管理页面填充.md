# 物联网设备管理页面填充

## 需求背景

当前物联网设备管理页仍是占位内容，无法展示设备接入规模、运行状态、告警处置和设备台账，进入该路由后也不能支撑演示或后续真实接口接入。

此外，上一次中断后页面入口文件缺失，当前 `/iot/equipment` 页面处于不完整状态，需要一并恢复。

## 目标与价值

- 为物联网设备管理页补齐可展示的中文业务内容
- 提供基础筛选交互，支持按关键字、区域、状态查看设备
- 让页面具备“总览 + 告警 + 台账”的信息结构，便于后续直接替换真实接口
- 恢复页面文件，避免路由访问时报错

## 功能范围

- 新增页面头部总览区，展示设备管理定位、重点状态和概览标签
- 新增四张概览指标卡，展示接入设备数、在线率、待处理告警数、远程指令成功率
- 新增设备台账区域，支持关键字搜索、区域筛选、状态筛选和一键重置
- 新增告警跟进卡片区，展示高优先级异常与推荐动作
- 新增区域运行卡片区，展示各区域在线完成度与异常数
- 所有页面文案使用中文

不在本次范围内：

- 不接入真实后端接口
- 不实现批量下发、新增接入等按钮的真实业务行为
- 不改动路由结构和菜单契约

## 用户操作流程

1. 用户进入“物联网设备管理”页面
2. 页面默认展示设备概览、重点告警、区域运行和完整设备列表
3. 用户可以输入设备名称、编号或类型关键字缩小列表范围
4. 用户可以按区域和设备状态进一步筛选
5. 用户点击“重置筛选”后，列表恢复到默认全量展示

## 验收标准

- 进入 `/iot/equipment` 页面时，可看到完整中文页面而不是占位内容
- 页面包含总览区、指标卡、设备台账、告警跟进、区域运行五类核心信息
- 关键字筛选可根据设备名称、编号或类型过滤列表
- 区域筛选和状态筛选可单独或组合生效
- 点击“重置筛选”后，关键字、区域、状态均恢复默认值
- 页面在桌面宽度下保持卡片式布局，在较窄宽度下可自然换行
